# Arry 的數位工具清單

> AI 寫作、推薦、評估工具時會翻這份清單。每當你換工具、加新服務、決定淘汰某個東西，就更新這裡。
> 這是第一版骨架，先填入目前從 `codex_installation` 環境能確認的工具；細節如「為什麼選它」與「已淘汰工具」仍待補充。

## 我用這些工具做這些事

### 通訊與信箱

- Gmail：主力信箱；Codex 可用 Gmail connector 協助搜尋、摘要與草擬回覆。

### 行事曆與排程

- Google Calendar：主力行事曆；Codex 可用 Google Calendar connector 查行程、找空檔、做會議準備。

### 筆記與知識管理

- Obsidian / Secondbrain：主要第二大腦，vault 位於 `/Users/arrywu/Library/CloudStorage/GoogleDrive-icestone0128@gmail.com/我的雲端硬碟/secondbrain`。
- Heptabase：可透過本機 `heptabase-cli` skill 互動；實際使用情境待補。
- NotebookLM：本機輸出資料夾預設為 `/Users/arrywu/Documents/NotebookLM`，依成品類型分資料夾管理。

### 程式開發與自動化

- Codex App：主要 AI 工作環境。
- GitHub：程式碼管理、repo、PR、部署狀態查詢。
- GitHub Pages：`codex_installation` 的網站部署來源為 `main` branch 的 `/docs` 目錄。
- Firebase：`codex_installation` 關聯專案為 `codex-4e80b`。

- Firecrawl：本地已跑成功，用於公開網頁抓取與研究資料擷取；API key 只保存在本機，不上傳 GitHub。

### 設計與內容創作

- Google Docs / Slides：文件與簡報協作。
- NotebookLM slides / infographics：作為 NotebookLM 成品輸出類型。
- Presentation workflow：用於 NotebookLM 簡報與 YAML 視覺控制工作流。

### 資料與追蹤

- Obsidian 知識庫：長期索引、log、每週知識重整。

## 我的工作流組合（誰串誰）

- Codex App -> Google Drive 專案資料夾：處理專案檔案、網頁、文件與設定。
- Codex App -> Obsidian Secondbrain：同步專案駕駛艙、知識重整、全域 skill 索引與長期工作規則。
- Codex App -> GitHub：提交、推送、PR、GitHub Pages 部署確認。
- NotebookLM -> `/Users/arrywu/Documents/NotebookLM`：下載或整理 NotebookLM 產物，再由 Codex 協助歸檔或轉成專案素材。
- Arry 助手資料層 -> 新專案：跨專案偏好與踩坑放在 `codex_installation/000_Agent`，專案本地只放該專案自己的 `100_Todo/` 與 `200_Reference/`。

## 維護節奏

每月 1 號花 5 分鐘檢查：

1. 上個月有裝新工具嗎？加進清單。
2. 上個月有淘汰工具嗎？移到「已淘汰」。
3. 有哪個工具說不出為什麼還在用？標成待評估。
