# 05-第二大腦設定指南

## 目標

把第二大腦整理成可長期使用的工作系統：剪藏、模板、知識庫、每日筆記、每週重整與專案駕駛艙。

## 前置條件

- 已完成 `04-建立第二大腦-Obsidian`。
- Obsidian Desktop 能看到 `{{OBSIDIAN_VAULT}}`。
- Codex 能讀寫 `{{OBSIDIAN_VAULT}}`。

## 必備資料夾

```text
{{OBSIDIAN_VAULT}}/
  AGENTS.md
  Clippings/
  知識庫/
    index.md
    log.md
  創作庫/
  每日筆記/
  Templates/
  專案庫/
```

## 建議模板

### 知識庫模板

```markdown
---
title: <標題>
date: <YYYY-MM-DD>
type: knowledge
tags:
  - 知識庫
---

# <標題>

## 核心摘要

## 來源

## 重點

## 可應用場景

## 待確認
```

### 每週重整模板

```markdown
---
title: <YYYY-Www> 知識重整
date: <YYYY-MM-DD>
type: weekly-review
tags:
  - 知識重整
---

# <YYYY-Www> 知識重整

## 本週新增

## 已整理

## 待確認

## 下一步
```

## Web Clipper 設定

Obsidian Web Clipper 要確認：

- vault 指向你的 vault 名稱，不是別人的範例名稱。
- clipping 預設資料夾可用 `Clippings/`。
- Desktop Obsidian 已開啟正確 vault。

## 每週知識重整

可以用 Codex automation 或手動請 Codex 執行。建議 prompt：

```text
讀取 {{OBSIDIAN_VAULT}}/AGENTS.md，針對本週新增的 Clippings、每日筆記與知識庫內容做一次補充式整理。不要覆寫既有筆記；如果需要更新 index.md 或 log.md，只做安全追加。最後列出新增/修改檔案與待確認事項。
```

若設成 automation，請確認：

- 工作區是 `{{OBSIDIAN_VAULT}}` 或可讀寫 vault 的專案。
- 頻率符合需求，例如每週日早上。
- prompt 不要求自動刪除或覆寫。

## 驗證

1. 新增一篇測試 clipping 到 `Clippings/`。
2. 請 Codex 依模板整理成一篇測試知識庫 note。
3. 確認 `index.md` 或 `log.md` 只被安全追加。
4. 刪除測試 note 前先確認路徑。

## 踩坑修正

- `Vault not found` 多半是 Web Clipper 的 vault 參數錯誤，不是檔案內容錯。
- Web Clipper 成功剪藏後，若 Desktop 看不到，先確認 Desktop vault 與 Web Clipper vault 是同一個。
- 每週重整寫到 `每日筆記/` 或專案駕駛艙，不要污染全域 `AGENTS.md`。
- `AGENTS.md` 寫固定規則；`log.md` 與駕駛艙寫事件與進度。
- 週期性自動化開始前，要先手動跑一次確認不會覆寫既有筆記。
