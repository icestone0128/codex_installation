## 新專案標準流程

1. 先檢查環境與現有資料夾。
2. 先問清楚專案名稱、用途、工作資料夾、GitHub、Obsidian、Firebase、部署目標。
3. 工作資料夾放 Google Drive，不放 Obsidian vault。
4. 專案根目錄建立或補齊：
   - `AGENTS.md`
   - `README.md`
   - `.gitignore`
   - Git repo
5. Obsidian 駕駛艙建立在：
   - `專案庫/<專案名稱>/專案工作流程.md`
6. 若需 GitHub，再建立 repo 與 remote。
7. 若需 Firebase，再建立或連接 project。

## 既有專案規則

若專案已存在：

- 先盤點已存在 / 缺少項目。
- 只補缺口。
- 不覆蓋既有 `AGENTS.md`、`README.md`、`.gitignore`、Firebase 設定或 Git 歷史。

## 命名規則

新專案資料夾建議使用：

- 小寫英文
- 數字
- 底線 `_`
- 連字號 `-`

避免空格。實測空格路徑曾造成補丁工具與側邊欄刷新困擾。
