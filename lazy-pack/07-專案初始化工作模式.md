## 新專案標準流程

1. 先檢查環境與現有資料夾。
2. 先問清楚專案名稱、用途、工作資料夾、GitHub、Obsidian、Firebase、部署目標。
3. 工作資料夾放 Google Drive，不放 Obsidian vault。
4. 專案根目錄建立或補齊：
   - `AGENTS.md`
   - `README.md`
   - `.gitignore`
   - Git repo
5. Obsidian 駕駛艙建立在：
   - `專案庫/<專案名稱>/專案工作流程.md`
6. 新專案預設整合 Arry 助手雙層資料層：
   - 專案 `AGENTS.md` 寫入 Arry 助手整合段落。
   - 使用全域核心層 `codex_installation/000_Agent`。
   - 專案本地建立 `100_Todo/` 與 `200_Reference/`。
   - 只有需要專案獨立助手記憶時，才建立 `000_Agent/project-memory/`。
   - 具體資料層以 `09-個人助手設定` 為準。
7. 若需 GitHub，再建立 repo 與 remote。
8. 若需 Firebase，再建立或連接 project。

## 既有專案規則

若專案已存在：

- 先盤點已存在 / 缺少項目。
- 只補缺口。
- 不覆蓋既有 `AGENTS.md`、`README.md`、`.gitignore`、Firebase 設定或 Git 歷史。
- 只補上 Arry 助手整合段落與缺少的本地 `100_Todo/`、`200_Reference/`，不複製或搬移全域核心資料層。

## 命名規則

新專案資料夾建議使用：

- 小寫英文
- 數字
- 底線 `_`
- 連字號 `-`

避免空格。實測空格路徑曾造成補丁工具與側邊欄刷新困擾。

- 新增全域 skills 後，要重開 Codex 對話或重啟 App。
- Arry 助手是 Codex App 全域 AI 分身；設定以 `09-個人助手設定` 為準。
- 來源文件若提到 `CLAUDE.md` 或 `~/.claude/skills`，要先轉成 Codex 的 `AGENTS.md` 與 `/Users/arrywu/.codex/skills`。
- Codex 側邊欄不會即時跟著 Finder / Google Drive 改名刷新。
- 專案改名時要同步改：
  - Google Drive 資料夾
  - `/Users/arrywu/.codex/config.toml`
  - 專案 `AGENTS.md`
  - Obsidian 專案庫資料夾
  - Obsidian 駕駛艙內文
  - Firebase MCP project directory
- GitHub 若沒有 remote，就不需要改 repo。
- Firebase Project ID 不會因本地專案改名而改變。
