# 09-個人助手設定

> 2026-06-21 更新：納入個人助手初始化分流、專案路徑索引與 `codex-skill-creator` 必要入口；本文件維持 Codex-only 路徑。


## 目標

把 來源工具 取向的 AI 分身資料夾思路，修正成 Codex App 可用的個人助手設定。

這份文件是之後設定個人助手與新專案資料層時的正式懶人包；舊的 Agent Folder 文檔只作為來源參考，不直接照做。

## 前置條件

- 已完成 `README.md` 的設定表。
- 已完成 `01-Codex-必裝-Skills-與-Plugins`。
- 若要同步到 Obsidian，已完成 `04-建立第二大腦-Obsidian` 與 `05-第二大腦設定指南`。
- 已決定個人助手名稱：`{{ASSISTANT_NAME}}`。
- 已決定設定專案位置：`{{SETUP_REPO}}`。
- 已決定 Obsidian vault：`{{OBSIDIAN_VAULT}}`。

## 固定結論

- 使用環境：Codex App。
- 不使用 來源工具 專用設定。
- Arry/個人助手 skill 是全域入口，位置固定在 `{{CODEX_HOME}}/skills/{{ASSISTANT_SKILL_NAME}}`；若已做跨裝置同步，這個資料夾才會 symlink 到 `{{SYNC_ROOT}}/skills`。
- 個人助手全域資料層：`{{ASSISTANT_ROOT}}`，預設為 `{{SYNC_ROOT}}`。
- 可攜式全域核心規則主檔：`{{ASSISTANT_ROOT}}/core-rules.md`；`{{CODEX_HOME}}/AGENTS.md` 只作為 Codex App 入口 symlink。
- 個人助手全域資料層固定包含 `core-rules.md`、`skills/`、`memories/`、`workflows/` 與 `knowledge/`。
- 專案自己的 assistant skill 或 workflow 放在該專案 `000_Agent/skills`，不要放進全域 skills，除非它已經成熟到需要跨所有專案觸發。
- 新建或修改 skill 時，一律先使用 `codex-skill-creator`，再判斷歸屬：跨專案可重用者放全域；只服務單一專案者放該專案 `000_Agent/skills`。
- 無論全域或專案 skill，都必須保留完整可攜式版本。
- 每個專案預設建立自己的工作盒、素材盒，以及需要時的 `000_Agent/skills` / `000_Agent/memories`。
- 專案本地記憶只使用 `000_Agent/memories/`。
- Obsidian 保留為長期知識、專案駕駛艙與既有每日筆記系統，不另外建立重複日記層。

建議命名：

```text
{{ASSISTANT_SKILL_NAME}} = <assistant-name>-assistant
```

例如：`my-assistant`。

## 初始化分流與路徑索引

建立個人助手前，先確認是「從零開始」、「已有專案」、「指定同步資料夾」或「自訂路徑」。先盤點現況，不在未確認前建檔或搬動 repo。

已有多個專案時，可選擇建立：

```text
{{ASSISTANT_ROOT}}/knowledge/REPOS.md
```

只記錄使用者確認的專案名稱、用途與本機路徑。專案進度、部署狀態與下一步仍以 Obsidian 專案駕駛艙為準，避免建立第二份狀態來源。

若下載者需要讀取 GitHub repo，先檢查 GitHub 登入與 repo 可見性；在驗證完成前不改檔、不 commit、不 push。

## 建立全域核心層

```bash
mkdir -p "{{ASSISTANT_ROOT}}"/{memories,workflows,knowledge}
touch "{{ASSISTANT_ROOT}}/core-rules.md"
```

建議建立：

```text
{{ASSISTANT_ROOT}}/core-rules.md
{{ASSISTANT_ROOT}}/memories/MEMORY.md
{{ASSISTANT_ROOT}}/workflows/README.md
{{ASSISTANT_ROOT}}/knowledge/README.md
```

`core-rules.md` 放跨工具共用的全域工作規則、固定路徑、同步邊界與長期操作規範；`memories/MEMORY.md` 放跨專案長期偏好與踩坑。不放專案私有資料或 secrets。

記憶 hook 路由規則：

- 正式 durable memory root 是 `{{ASSISTANT_ROOT}}/memories`。
- `{{CODEX_HOME}}/memories` 只有在確認為指向 `{{ASSISTANT_ROOT}}/memories` 的 symlink 時，才可視為同一份記憶的本機入口。
- 寫入跨專案偏好、踩坑或 ad-hoc memory note 前，先解析 symlink；若 `{{CODEX_HOME}}/memories` 不是 symlink 到 `{{ASSISTANT_ROOT}}/memories`，不得把個人助手記憶寫入該本機路徑。
- 優先直接寫入 `{{ASSISTANT_ROOT}}/memories/extensions/ad_hoc/notes/` 或 `{{ASSISTANT_ROOT}}/memories/MEMORY.md`，避免 agent 誤判記憶位置。

## 建立個人助手全域 Skill

如果希望 Codex App 在任何專案都能用同一個助手，建立一個全域入口 skill，放在：

```text
{{CODEX_HOME}}/skills/{{ASSISTANT_SKILL_NAME}}/SKILL.md
```

如果要先使用本懶人包內的 `arry-assistant` 模板，可先複製，再把資料夾名稱、frontmatter `name` 與本文中的 `{{ASSISTANT_NAME}}` / `{{ASSISTANT_ROOT}}` 改成自己的值：

```bash
mkdir -p "{{CODEX_HOME}}/skills/{{ASSISTANT_SKILL_NAME}}"
# 請使用本文文末「內建 Skill 完整安裝內容」；不需要額外複製舊版獨立 skills 子目錄。
test -f "{{CODEX_HOME}}/skills/{{ASSISTANT_SKILL_NAME}}/SKILL.md" && echo "{{ASSISTANT_SKILL_NAME}} installed"
```

最小範本：

```markdown
---
name: {{ASSISTANT_SKILL_NAME}}
description: Use when the user asks to use {{ASSISTANT_NAME}}, personal assistant memory, cross-project preferences, or reusable assistant workflows. Read the shared global layer in {{ASSISTANT_ROOT}} and keep project-specific drafts inside the current project.
metadata:
  short-description: Use personal assistant context
---

# {{ASSISTANT_NAME}}

## Default Paths

- Shared global layer: `{{ASSISTANT_ROOT}}`
- Obsidian vault: `{{OBSIDIAN_VAULT}}`

## Rules

1. Read project `AGENTS.md` first.
2. Use shared core memory only for cross-project preferences.
3. Keep project drafts and references in the current project.
4. Do not write secrets or unnecessary personal data.
```

## 個人助手 Workflows 與專案本地 Skills

只服務個人助手核心、尚未成熟為全域 skill 的跨專案流程，不放 `{{CODEX_HOME}}/skills`。請先放在：

```text
{{ASSISTANT_ROOT}}/workflows/<local-workflow-name>/
```

或在專案內放：

```text
<project-root>/000_Agent/skills/<local-skill-name>/
```

這類全域 workflow 草稿可放在 `{{ASSISTANT_ROOT}}/workflows`。等到它需要跨所有專案使用時，再升級到 `{{CODEX_HOME}}/skills/<skill-name>`，並同步 LazyPack 與 Obsidian 全域 Skills 索引。

專案本地 skill 也要可攜化：`000_Agent/skills/<local-skill-name>/` 內至少要有 `SKILL.md`，需要時加入 `references/`、`scripts/`、`assets/`。若專案使用 Git，這個資料夾應跟著專案提交；若專案不使用 Git，至少在專案駕駛艙記錄 skill 名稱、位置、用途與同步狀態。

## 專案本地資料層

新專案或既有專案要補齊：

```text
100_Todo/
  drafts/
  projects/
  archive/

200_Reference/
  writing-samples/
  past-work/
  templates/
```

預設只延伸到上述一階資料夾；除非使用者明確要求，不再為內容類型建立更深層分類。

只有當某個專案真的需要獨立助手記憶時，才建立：

```text
000_Agent/skills/
000_Agent/memories/
```

## Obsidian 同步

個人助手相關固定筆記可放：

```text
{{OBSIDIAN_PROJECTS}}/{{SETUP_PROJECT_NAME}}/{{ASSISTANT_NAME}}/
```

新專案初始化時，專案駕駛艙放在：

```text
{{OBSIDIAN_PROJECTS}}/<專案名稱>/專案工作流程.md
```

需要寫入駕駛艙的內容：

- 專案名稱、用途、工作資料夾。
- GitHub / Firebase / 部署狀態。
- 個人助手整合段落。
- 本地 `100_Todo/`、`200_Reference/` 的用途。
- 是否有專案專屬 `000_Agent/skills/` 或 `000_Agent/memories/`。

## 驗證

1. `{{ASSISTANT_ROOT}}/memories/`、`{{ASSISTANT_ROOT}}/workflows/` 與 `{{ASSISTANT_ROOT}}/skills/` 存在。
2. 若使用全域助手入口，`{{CODEX_HOME}}/skills/{{ASSISTANT_SKILL_NAME}}/SKILL.md` 存在。
3. 新開 Codex 對話後，要求使用 `{{ASSISTANT_NAME}}`。
4. Codex 能讀 `{{ASSISTANT_ROOT}}`。
5. 新專案初始化時能建立 `100_Todo/`、`200_Reference/`，並在需要本地 assistant skill 時建立 `000_Agent/skills/`。

## 踩坑修正

- 舊文檔是 來源工具 教學，不能直接照搬到 Codex App。
- 來源工具規則檔 不會被 Codex App 當作主要規則檔；Codex 要使用 `AGENTS.md` 與 skill。
- Codex 全域 skills 不放在 來源工具的 skills 路徑；需要全域觸發時放 `{{CODEX_HOME}}/skills`。
- 專案或助手本地 skills 放 `000_Agent/skills`；不要把它 symlink 到 `{{CODEX_HOME}}/skills`。
- 不要建立 來源工具 symlink；這會讓 Codex 設定變複雜，也會混淆未來維護。
- 新增或修改全域 skill 後，通常要開新對話或重啟 Codex App 才會穩定載入。
- 個人助手全域核心層只放一份，不要複製到每個專案。
- 專案資料夾與 Obsidian vault 分開；工作資料夾預設放 `{{WORK_ROOT}}`，Obsidian 只放駕駛艙與知識筆記。
- 既有專案只補缺口，不覆蓋 `AGENTS.md`、`README.md`、`.gitignore`、Git 歷史、Firebase 設定。
- 不把 `.env`、API key、token、密碼、Admin 憑證、個資或敏感資料寫入 repo 或筆記。

# Arry 助手

Arry 助手是使用者的 Codex App 全域 AI 分身。正式設定以 `codex_installation/lazy-pack/09-個人助手設定.md` 為準。

This skill is a personal-assistant template. When copying it for another user, rename the folder and `name` if desired, then replace `ASSISTANT_NAME`, `ASSISTANT_ROOT`, `OBSIDIAN_VAULT`, output folders, writing samples, and sync paths before using it for real memory or note writes.

## 固定定位

- 分身名稱：Arry 助手。
- 使用環境：Codex App。
- 使用者角色：科技業一線主管。
- 優先協助任務：資料研究與知識管理。
- 主要輸出：Facebook 貼文、Email/客戶溝通、書籍或 Podcast 的讀後心得。
- 不使用 來源工具 專用設定：不要建立或依賴 `來源工具舊規則檔`、`來源工具的舊 skills 路徑`、來源工具 slash command shim。
- Codex 全域 skills 正式位置：`{{CODEX_HOME}}/skills`。
- Arry 助手全域資料層：`{{SYNC_ROOT}}`。
- Arry 助手本身是全域入口 skill，專案初始化時預設帶入。
- Arry 助手全域資料層固定包含 `skills/`、`memories/`、`workflows/` 與 `knowledge/`。
- Codex 全域 skills：`{{SYNC_ROOT}}/skills`；`{{CODEX_HOME}}/skills` symlink 指向這裡。
- Arry 助手跨專案記憶：`{{SYNC_ROOT}}/memories/MEMORY.md`。
- Arry 助手跨專案 workflow 草稿：`{{SYNC_ROOT}}/workflows`。
- 個人助手設定：`{{SETUP_REPO}}/lazy-pack/09-個人助手設定.md`。
- Obsidian 同步位置：`{{OBSIDIAN_PROJECTS}}/{{SETUP_PROJECT_NAME}}/Arry 助手`。

Portable placeholders:

- `ASSISTANT_NAME`: the personal assistant name.
- `ASSISTANT_ROOT`: the global assistant asset root that contains `skills/`, `memories/`, `workflows/`, and `knowledge/`.
- `OBSIDIAN_VAULT`: the user's Obsidian vault, if used.
- `PROJECT_LIBRARY`: the vault-relative project cockpit folder, if used.

## 使用流程

1. 先讀取專案 `AGENTS.md`，若任務涉及筆記或同步，再讀取 Obsidian vault 的 `AGENTS.md`。
2. 讀取個人助手記憶：`{{SYNC_ROOT}}/memories/MEMORY.md`；可攜化設定中等同於 `ASSISTANT_ROOT/memories/MEMORY.md`。
3. 若任務涉及新專案初始化，遵守既有 `project-init-sync` 規則，並參照 `09-個人助手設定`；不要讓 Arry 助手資料層覆蓋新專案標準流程。
4. 若來源文件是 來源工具 教學，轉成 Codex App 版本：
   - `來源工具舊規則檔` 規則改寫到 Codex 可讀的 `AGENTS.md` 或全域 skill。
   - `來源工具的舊 skills 路徑` 改成 `{{CODEX_HOME}}/skills`。
   - 來源工具 command / symlink / terminal-only 步驟只保留為參考，除非使用者明確要支援 來源工具。
5. 若新增、修改、刪除全域 skill，更新 Obsidian 的全域 Skills 同步筆記。
6. 若使用者要新增 skill，先判斷歸屬：
   - 一律先使用全域 `codex-skill-creator` 進行建立、改良、驗證與同步；不繞過該 skill 直接交給其他 creator。
   - 跨專案可重用或需要全域觸發：放 `{{CODEX_HOME}}/skills/<skill-name>`，並同步 `{{SETUP_REPO}}/lazy-pack/<對應序號文件>` 與 Obsidian 全域 Skills 索引。
   - 只服務目前專案：放 `<project-root>/000_Agent/skills/<skill-name>`，保留完整可攜式 skill 包並記錄到專案駕駛艙。

## 初始化與專案路徑索引

- 在幫使用者建立或重整個人助手資料層前，先判斷是「從零開始」、「已有專案」、「指定同步資料夾」或「自訂路徑」；確認後才建檔。
- 使用者已有多個專案時，可在 `{{SYNC_ROOT}}/knowledge/REPOS.md` 維護經使用者確認的專案名稱、用途與本機路徑。
- `REPOS.md` 只作穩定路徑索引；目前進度、部署狀態與下一步仍放專案 Obsidian 駕駛艙，避免雙重狀態來源。
- 不搬動或複製既有 repo 來建立索引；掃描結果只能當候選清單，寫入前要讓使用者確認。

## 輸出路由

- Facebook 貼文：草稿放 `100_Todo/drafts/`，範例讀 `200_Reference/writing-samples/`。
- Email/客戶溝通：草稿放 `100_Todo/drafts/`，範例讀 `200_Reference/writing-samples/`，模板讀 `200_Reference/templates/`。
- 書籍或 Podcast 讀後心得：草稿放 `100_Todo/drafts/`。範例優先讀 `200_Reference/writing-samples/`。
- 資料研究與知識管理輸出：先整理來源、脈絡、關鍵發現、可用行動，再判斷要寫入專案本地 `200_Reference/`、Obsidian 知識庫或專案駕駛艙。

## 雙層專案整合模式

- 預設採用 `09-個人助手設定` 的四盒結構：助手核心盒、行動工作盒、參考素材盒、Obsidian 知識盒。
- 全域共用層：`codex_symlink/`，存放 Arry 助手核心記憶、跨專案偏好、踩坑、全域 skills 與 workflow 草稿。
- 專案本地層：每個專案建立自己的 `100_Todo/` 與 `200_Reference/`，存放該專案的草稿、進行中事項、素材、範例與模板。
- 專案本地任務目錄只延伸到：`100_Todo/drafts/`、`100_Todo/projects/`、`100_Todo/archive/`。
- 專案本地參考目錄只延伸到：`200_Reference/writing-samples/`、`200_Reference/templates/`、`200_Reference/past-work/`。
- 專案本地 skill 層：只有當專案需要自己的 assistant skill 或局部流程時，建立 `<project-root>/000_Agent/skills`。
- Obsidian 知識層：專案駕駛艙與既有每日筆記系統留在 Obsidian。
- 專案本地記憶：只有在專案需要獨立助手記憶時，才建立 `000_Agent/memories/`。
- 未來新專案：由 `project-init-sync` 在專案 `AGENTS.md` 與 Obsidian 駕駛艙加入雙層資料層說明，並補齊本地資料夾。
- 現有專案：補上整合段落與本地 `100_Todo/`、`200_Reference/`，不複製或搬移全域核心層。
- 專案自己的進度、下一步、部署狀態留在該專案駕駛艙。
