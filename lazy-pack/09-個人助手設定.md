# 09-個人助手設定

## 目標

把 來源工具 取向的 AI 分身資料夾思路，修正成 Codex App 可用的個人助手設定。

這份文件是之後設定個人助手與新專案資料層時的正式懶人包；舊的 Agent Folder 文檔只作為來源參考，不直接照做。

## 前置條件

- 已完成 `README.md` 的設定表。
- 已完成 `01-Codex-必裝-Skills-與-Plugins`。
- 若要同步到 Obsidian，已完成 `04-建立第二大腦-Obsidian` 與 `05-第二大腦設定指南`。
- 已決定個人助手名稱：`{{ASSISTANT_NAME}}`。
- 已決定設定專案位置：`{{SETUP_REPO}}`。
- 已決定 Obsidian vault：`{{OBSIDIAN_VAULT}}`。

## 固定結論

- 使用環境：Codex App。
- 不使用 來源工具 專用設定。
- Arry/個人助手 skill 是全域入口，位置固定在 `{{CODEX_HOME}}/skills/{{ASSISTANT_SKILL_NAME}}`；若已做跨裝置同步，這個資料夾才會 symlink 到 `{{SYNC_ROOT}}/skills`。
- 個人助手全域資料層：`{{ASSISTANT_ROOT}}`，預設為 `{{SYNC_ROOT}}`。
- 個人助手全域資料層固定包含 `skills/`、`memory/`、`workflows/` 與 `knowledge/`。
- 專案自己的 assistant skill 或 workflow 放在該專案 `000_Agent/skills`，不要放進全域 skills，除非它已經成熟到需要跨所有專案觸發。
- 新建 skill 時，先由 Arry 助手判斷歸屬：跨專案可重用者放全域；只服務單一專案者放該專案 `000_Agent/skills`。
- 無論全域或專案 skill，都必須保留完整可攜式版本。
- 每個專案預設建立自己的工作盒、素材盒，以及需要時的 `000_Agent/skills` / `000_Agent/memory`。
- 專案本地記憶只使用 `000_Agent/memory/`。
- Obsidian 保留為長期知識、專案駕駛艙與既有每日筆記系統，不另外建立重複日記層。

建議命名：

```text
{{ASSISTANT_SKILL_NAME}} = <assistant-name>-assistant
```

例如：`my-assistant`。

## 四盒結構

| 盒子 | 位置 | 用途 |
| --- | --- | --- |
| 1. 助手核心盒 | `{{ASSISTANT_ROOT}}/memory`、`{{ASSISTANT_ROOT}}/workflows`、`{{ASSISTANT_ROOT}}/knowledge` | 個人助手核心記憶、跨專案偏好、踩坑、角色設定與 workflow 草稿 |
| 2. 行動工作盒 | 各專案 `100_Todo/` | 草稿、待辦、進行中任務、行動清單 |
| 3. 參考素材盒 | 各專案 `200_Reference/` | 寫作範例、模板、過去作品、研究素材 |
| 4. Obsidian 知識盒 | `{{OBSIDIAN_PROJECTS}}/<專案名稱>/專案工作流程.md` 與既有知識庫 | 專案駕駛艙、長期知識、既有每日筆記同步 |

## 舊 Agent Folder 文檔的轉換規則

| 舊文檔項目 | Codex App 修正版 |
| --- | --- |
| 來源工具規則檔 | 改寫到專案 `AGENTS.md`、全域 `{{CODEX_HOME}}/AGENTS.md` 或對應 skill |
| 來源工具的全域 skills 路徑 | 需要全域觸發時改用 `{{CODEX_HOME}}/skills` |
| 來源工具的專案級 skills | 改放該專案 `000_Agent/skills`，只服務該專案 |
| 專案內 `000_Agent/skills` symlink | 不做 symlink；它是專案/助手本地 skill 區，不等於 Codex 全域 skills |
| 來源工具 slash command shim | 不建立；除非使用者明確要求支援來源工具 |
| 來源文件預設 AI 名稱 | 改成 `{{ASSISTANT_NAME}}` |
| `300_Journal/` | 不建立新的重複日記資料夾 |
| 「明天作業清單」 | 改為「行動清單」 |

## 建立全域核心層

```bash
mkdir -p "{{ASSISTANT_ROOT}}"/{memory,workflows,knowledge}
```

建議建立：

```text
{{ASSISTANT_ROOT}}/memory/MEMORY.md
{{ASSISTANT_ROOT}}/workflows/README.md
{{ASSISTANT_ROOT}}/knowledge/README.md
```

內容只放跨專案長期偏好與踩坑，不放專案私有資料。

## 建立個人助手全域 Skill

如果希望 Codex App 在任何專案都能用同一個助手，建立一個全域入口 skill，放在：

```text
{{CODEX_HOME}}/skills/{{ASSISTANT_SKILL_NAME}}/SKILL.md
```

如果要先使用本懶人包內的 `arry-assistant` 模板，可先複製，再把資料夾名稱、frontmatter `name` 與本文中的 `{{ASSISTANT_NAME}}` / `{{ASSISTANT_ROOT}}` 改成自己的值：

```bash
mkdir -p "{{CODEX_HOME}}/skills/{{ASSISTANT_SKILL_NAME}}"
rsync -a --delete "{{SETUP_REPO}}/lazy-pack/skills/arry-assistant/" "{{CODEX_HOME}}/skills/{{ASSISTANT_SKILL_NAME}}/"
test -f "{{CODEX_HOME}}/skills/{{ASSISTANT_SKILL_NAME}}/SKILL.md" && echo "{{ASSISTANT_SKILL_NAME}} installed"
```

最小範本：

```markdown
---
name: {{ASSISTANT_SKILL_NAME}}
description: Use when the user asks to use {{ASSISTANT_NAME}}, personal assistant memory, cross-project preferences, or reusable assistant workflows. Read the shared global layer in {{ASSISTANT_ROOT}} and keep project-specific drafts inside the current project.
metadata:
  short-description: Use personal assistant context
---

# {{ASSISTANT_NAME}}

## Default Paths

- Shared global layer: `{{ASSISTANT_ROOT}}`
- Obsidian vault: `{{OBSIDIAN_VAULT}}`

## Rules

1. Read project `AGENTS.md` first.
2. Use shared core memory only for cross-project preferences.
3. Keep project drafts and references in the current project.
4. Do not write secrets or unnecessary personal data.
```

## 個人助手 Workflows 與專案本地 Skills

只服務個人助手核心、尚未成熟為全域 skill 的跨專案流程，不放 `{{CODEX_HOME}}/skills`。請先放在：

```text
{{ASSISTANT_ROOT}}/workflows/<local-workflow-name>/
```

或在專案內放：

```text
<project-root>/000_Agent/skills/<local-skill-name>/
```

這類全域 workflow 草稿可放在 `{{ASSISTANT_ROOT}}/workflows`。等到它需要跨所有專案使用時，再升級到 `{{CODEX_HOME}}/skills/<skill-name>`，並同步 LazyPack 與 Obsidian 全域 Skills 索引。

專案本地 skill 也要可攜化：`000_Agent/skills/<local-skill-name>/` 內至少要有 `SKILL.md`，需要時加入 `references/`、`scripts/`、`assets/`。若專案使用 Git，這個資料夾應跟著專案提交；若專案不使用 Git，至少在專案駕駛艙記錄 skill 名稱、位置、用途與同步狀態。

## 專案本地資料層

新專案或既有專案要補齊：

```text
100_Todo/
  drafts/
  projects/
  archive/

200_Reference/
  writing-samples/
  past-work/
  templates/
```

預設只延伸到上述一階資料夾；除非使用者明確要求，不再為內容類型建立更深層分類。

只有當某個專案真的需要獨立助手記憶時，才建立：

```text
000_Agent/skills/
000_Agent/memory/
```

## Obsidian 同步

個人助手相關固定筆記可放：

```text
{{OBSIDIAN_PROJECTS}}/{{SETUP_PROJECT_NAME}}/{{ASSISTANT_NAME}}/
```

新專案初始化時，專案駕駛艙放在：

```text
{{OBSIDIAN_PROJECTS}}/<專案名稱>/專案工作流程.md
```

需要寫入駕駛艙的內容：

- 專案名稱、用途、工作資料夾。
- GitHub / Firebase / 部署狀態。
- 個人助手整合段落。
- 本地 `100_Todo/`、`200_Reference/` 的用途。
- 是否有專案專屬 `000_Agent/skills/` 或 `000_Agent/memory/`。

## 驗證

1. `{{ASSISTANT_ROOT}}/memory/`、`{{ASSISTANT_ROOT}}/workflows/` 與 `{{ASSISTANT_ROOT}}/skills/` 存在。
2. 若使用全域助手入口，`{{CODEX_HOME}}/skills/{{ASSISTANT_SKILL_NAME}}/SKILL.md` 存在。
3. 新開 Codex 對話後，要求使用 `{{ASSISTANT_NAME}}`。
4. Codex 能讀 `{{ASSISTANT_ROOT}}`。
5. 新專案初始化時能建立 `100_Todo/`、`200_Reference/`，並在需要本地 assistant skill 時建立 `000_Agent/skills/`。

## 設定範例

範例使用「Arry 助手」作為個人助手名稱。下載者請改成自己的助手名稱。

## 踩坑修正

- 舊文檔是 來源工具 教學，不能直接照搬到 Codex App。
- 來源工具規則檔 不會被 Codex App 當作主要規則檔；Codex 要使用 `AGENTS.md` 與 skill。
- Codex 全域 skills 不放在 來源工具的 skills 路徑；需要全域觸發時放 `{{CODEX_HOME}}/skills`。
- 專案或助手本地 skills 放 `000_Agent/skills`；不要把它 symlink 到 `{{CODEX_HOME}}/skills`。
- 不要建立 來源工具 symlink；這會讓 Codex 設定變複雜，也會混淆未來維護。
- 新增或修改全域 skill 後，通常要開新對話或重啟 Codex App 才會穩定載入。
- 個人助手全域核心層只放一份，不要複製到每個專案。
- 專案資料夾與 Obsidian vault 分開；工作資料夾預設放 `{{WORK_ROOT}}`，Obsidian 只放駕駛艙與知識筆記。
- 既有專案只補缺口，不覆蓋 `AGENTS.md`、`README.md`、`.gitignore`、Git 歷史、Firebase 設定。
- 不把 `.env`、API key、token、密碼、Admin 憑證、個資或敏感資料寫入 repo 或筆記。
