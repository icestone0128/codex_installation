# 09-個人助手設定

## 目標

把 來源工具 取向的 AI 分身資料夾思路，修正成 Codex App 可用的個人助手設定。

這份文件是之後設定個人助手與新專案資料層時的正式懶人包；舊的 Agent Folder 文檔只作為來源參考，不直接照做。

## 前置條件

- 已完成 `README.md` 的設定表。
- 已完成 `01-Codex-必裝-Skills-與-Plugins`。
- 若要同步到 Obsidian，已完成 `04-建立第二大腦-Obsidian` 與 `05-第二大腦設定指南`。
- 已決定個人助手名稱：`{{ASSISTANT_NAME}}`。
- 已決定設定專案位置：`{{SETUP_REPO}}`。
- 已決定 Obsidian vault：`{{OBSIDIAN_VAULT}}`。

## 固定結論

- 使用環境：Codex App。
- 不使用 來源工具 專用設定。
- 全域 skill 位置：`{{CODEX_HOME}}/skills/{{ASSISTANT_SKILL_NAME}}`。
- 個人助手全域核心資料層：`{{SETUP_REPO}}/000_Agent`。
- 每個專案只建立自己的工作盒與素材盒。
- Obsidian 保留為長期知識、專案駕駛艙與既有每日筆記系統，不另外建立重複日記層。

建議命名：

```text
{{ASSISTANT_SKILL_NAME}} = <assistant-name>-assistant
```

例如：`my-assistant`。

## 四盒結構

| 盒子 | 位置 | 用途 |
| --- | --- | --- |
| 1. 助手核心盒 | `{{SETUP_REPO}}/000_Agent` | 個人助手核心記憶、跨專案偏好、踩坑、角色設定 |
| 2. 行動工作盒 | 各專案 `100_Todo/` | 草稿、待辦、進行中任務、行動清單 |
| 3. 參考素材盒 | 各專案 `200_Reference/` | 寫作範例、模板、過去作品、研究素材 |
| 4. Obsidian 知識盒 | `{{OBSIDIAN_PROJECTS}}/<專案名稱>/專案工作流程.md` 與既有知識庫 | 專案駕駛艙、長期知識、既有每日筆記同步 |

## 舊 Agent Folder 文檔的轉換規則

| 舊文檔項目 | Codex App 修正版 |
| --- | --- |
| 來源工具規則檔 | 改寫到專案 `AGENTS.md`、全域 `{{CODEX_HOME}}/AGENTS.md` 或對應 skill |
| 來源工具的 skills 路徑 | 改用 `{{CODEX_HOME}}/skills` |
| 專案內 `000_Agent/skills` symlink | 不做 symlink；正式 skill 放 Codex 全域 skills |
| 來源工具 slash command shim | 不建立；除非使用者明確要求支援來源工具 |
| 來源文件預設 AI 名稱 | 改成 `{{ASSISTANT_NAME}}` |
| `300_Journal/` | 不建立新的重複日記資料夾 |
| 「明天作業清單」 | 改為「行動清單」 |

## 建立全域核心層

```bash
mkdir -p "{{SETUP_REPO}}/000_Agent"/{memory,workflows}
```

建議建立：

```text
{{SETUP_REPO}}/000_Agent/profile.md
{{SETUP_REPO}}/000_Agent/preferences.md
{{SETUP_REPO}}/000_Agent/pitfalls.md
```

內容只放跨專案長期偏好與踩坑，不放專案私有資料。

## 建立個人助手 Skill

放在：

```text
{{CODEX_HOME}}/skills/{{ASSISTANT_SKILL_NAME}}/SKILL.md
```

如果要先使用本懶人包內的 `arry-assistant` 模板，可先複製，再把資料夾名稱、frontmatter `name` 與本文中的 `{{ASSISTANT_NAME}}` / `{{ASSISTANT_ROOT}}` 改成自己的值：

```bash
mkdir -p "{{CODEX_HOME}}/skills/{{ASSISTANT_SKILL_NAME}}"
rsync -a --delete "{{SETUP_REPO}}/lazy-pack/skills/arry-assistant/" "{{CODEX_HOME}}/skills/{{ASSISTANT_SKILL_NAME}}/"
test -f "{{CODEX_HOME}}/skills/{{ASSISTANT_SKILL_NAME}}/SKILL.md" && echo "{{ASSISTANT_SKILL_NAME}} installed"
```

最小範本：

```markdown
---
name: {{ASSISTANT_SKILL_NAME}}
description: Use when the user asks to use {{ASSISTANT_NAME}}, personal assistant memory, cross-project preferences, or reusable assistant workflows. Read the shared core layer in {{SETUP_REPO}}/000_Agent and keep project-specific drafts inside the current project.
metadata:
  short-description: Use personal assistant context
---

# {{ASSISTANT_NAME}}

## Default Paths

- Shared core layer: `{{SETUP_REPO}}/000_Agent`
- Obsidian vault: `{{OBSIDIAN_VAULT}}`

## Rules

1. Read project `AGENTS.md` first.
2. Use shared core memory only for cross-project preferences.
3. Keep project drafts and references in the current project.
4. Do not write secrets or unnecessary personal data.
```

## 專案本地資料層

新專案或既有專案要補齊：

```text
100_Todo/
  drafts/
  projects/
  archive/

200_Reference/
  writing-samples/
  past-work/
  templates/
```

依個人用途可加：

```text
100_Todo/drafts/social-posts/
100_Todo/drafts/emails/
100_Todo/drafts/articles/
100_Todo/drafts/scripts/

200_Reference/writing-samples/social/
200_Reference/writing-samples/emails/
200_Reference/writing-samples/articles/
200_Reference/templates/
```

只有當某個專案真的需要獨立助手記憶時，才建立：

```text
000_Agent/project-memory/
```

## Obsidian 同步

個人助手相關固定筆記可放：

```text
{{OBSIDIAN_PROJECTS}}/{{SETUP_PROJECT_NAME}}/{{ASSISTANT_NAME}}/
```

新專案初始化時，專案駕駛艙放在：

```text
{{OBSIDIAN_PROJECTS}}/<專案名稱>/專案工作流程.md
```

需要寫入駕駛艙的內容：

- 專案名稱、用途、工作資料夾。
- GitHub / Firebase / 部署狀態。
- 個人助手整合段落。
- 本地 `100_Todo/`、`200_Reference/` 的用途。
- 是否有專案專屬 `000_Agent/project-memory/`。

## 驗證

1. `{{CODEX_HOME}}/skills/{{ASSISTANT_SKILL_NAME}}/SKILL.md` 存在。
2. 新開 Codex 對話後，要求使用 `{{ASSISTANT_NAME}}`。
3. Codex 能讀 `{{SETUP_REPO}}/000_Agent`。
4. 新專案初始化時能建立 `100_Todo/`、`200_Reference/`。

## 設定範例

範例使用「Arry 助手」作為個人助手名稱。下載者請改成自己的助手名稱。

## 踩坑修正

- 舊文檔是 來源工具 教學，不能直接照搬到 Codex App。
- 來源工具規則檔 不會被 Codex App 當作主要規則檔；Codex 要使用 `AGENTS.md` 與 skill。
- Codex skills 不放在 來源工具的 skills 路徑，要放 `{{CODEX_HOME}}/skills`。
- 不要建立 來源工具 symlink；這會讓 Codex 設定變複雜，也會混淆未來維護。
- 新增或修改全域 skill 後，通常要開新對話或重啟 Codex App 才會穩定載入。
- 個人助手全域核心層只放一份，不要複製到每個專案。
- 專案資料夾與 Obsidian vault 分開；工作資料夾預設放 `{{WORK_ROOT}}`，Obsidian 只放駕駛艙與知識筆記。
- 既有專案只補缺口，不覆蓋 `AGENTS.md`、`README.md`、`.gitignore`、Git 歷史、Firebase 設定。
- 不把 `.env`、API key、token、密碼、Admin 憑證、個資或敏感資料寫入 repo 或筆記。
