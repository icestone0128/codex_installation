# 09-個人助手設定

## 目標

把原本 `01-agent-folder-setup.md` 裡的 AI 分身資料夾思路，修正成 Codex App 可用的 Arry 助手設定。

這份文件是之後設定 Arry 助手與新專案資料層時的正式懶人包；舊的 Agent Folder 文檔只作為來源參考，不再直接照做。

## 固定結論

- AI 分身名稱：Arry 助手。
- 使用環境：Codex App。
- 不使用 Claude Code 專用設定。
- 全域 skill 位置：`/Users/arrywu/.codex/skills/arry-assistant`
- Arry 助手全域核心資料層：`codex_installation/000_Agent`
- 每個專案只建立自己的工作盒與素材盒。
- Obsidian 保留為長期知識、專案駕駛艙與既有每日筆記系統，不另外建立重複日記層。

## 四盒結構

Codex 版 Arry 助手採用四盒，但第四盒不是舊文檔的 `300_Journal/`。

| 盒子 | 位置 | 用途 |
| --- | --- | --- |
| 1. 助手核心盒 | `codex_installation/000_Agent` | Arry 助手核心記憶、跨專案偏好、踩坑、角色設定 |
| 2. 行動工作盒 | 各專案 `100_Todo/` | 草稿、待辦、進行中任務、行動清單 |
| 3. 參考素材盒 | 各專案 `200_Reference/` | 寫作範例、模板、過去作品、研究素材 |
| 4. Obsidian 知識盒 | `secondbrain/專案庫/<專案名稱>/專案工作流程.md` 與既有知識庫 | 專案駕駛艙、長期知識、既有每日筆記同步 |

## 舊 Agent Folder 文檔的轉換規則

| 舊文檔項目 | Codex App 修正版 |
| --- | --- |
| `CLAUDE.md` | 改寫到專案 `AGENTS.md`、全域 `/Users/arrywu/.codex/AGENTS.md` 或對應 skill |
| `~/.claude/skills` | 改用 `/Users/arrywu/.codex/skills` |
| 專案內 `000_Agent/skills` symlink | 不做 symlink；正式 skill 放 Codex 全域 skills |
| Claude slash command shim | 不建立；除非使用者明確要求支援 Claude |

| `300_Journal/` | 不建立新的重複日記資料夾 |

## Arry 助手目前角色設定

- 使用者角色：科技業一線主管。
- 優先協助任務：資料研究與知識管理。
- 主要輸出平台與格式：
  - Facebook 貼文
  - Email / 客戶溝通
  - 書籍或 Podcast 讀後心得

## 專案本地資料層

新專案或既有專案要補齊的本地資料夾：

```text
100_Todo/
  drafts/
  projects/
  archive/

200_Reference/
  writing-samples/
  past-work/
  templates/
```

依 Arry 助手目前用途，常見子資料夾：

```text
100_Todo/drafts/social-posts/
100_Todo/drafts/emails/
100_Todo/drafts/articles/
100_Todo/drafts/scripts/

200_Reference/writing-samples/social/
200_Reference/writing-samples/emails/
200_Reference/writing-samples/articles/
200_Reference/writing-samples/scripts/
200_Reference/templates/email-templates/
```

只有當某個專案真的需要獨立助手記憶時，才建立：

```text
000_Agent/project-memory/
```

這個資料夾只放該專案專屬記憶，不複製或搬移全域 `codex_installation/000_Agent`。

## Obsidian 同步

Arry 助手相關固定筆記位置：

```text
secondbrain/專案庫/codex_installation/Arry 助手/
```

新專案初始化時，專案駕駛艙放在：

```text
secondbrain/專案庫/<專案名稱>/專案工作流程.md
```

需要寫入駕駛艙的內容：

- 專案名稱、用途、工作資料夾。
- GitHub / Firebase / 部署狀態。
- Arry 助手整合段落。
- 本地 `100_Todo/`、`200_Reference/` 的用途。
- 是否有專案專屬 `000_Agent/project-memory/`。

既有 Obsidian 每日筆記系統照原本方式運作，不因 Arry 助手設定而改名、搬移或重建。

## 踩坑修正

- 舊文檔是 Claude Code 教學，不能直接照搬到 Codex App。
- `CLAUDE.md` 不會被 Codex App 當作主要規則檔；Codex 要使用 `AGENTS.md` 與 skill。
- Codex skills 不放在 `~/.claude/skills`，要放 `/Users/arrywu/.codex/skills`。
- 不要建立 Claude symlink；這會讓 Codex 設定變複雜，也會混淆未來維護。
- 新增或修改全域 skill 後，通常要開新對話或重啟 Codex App 才會穩定載入。
- 全域 skill 有變更時，要同步更新 Obsidian 的全域 Skills 同步筆記。
- Arry 助手全域核心層只放一份，不要複製到每個專案。
- 專案資料夾與 Obsidian vault 分開；工作資料夾預設放 Google Drive，Obsidian 只放駕駛艙與知識筆記。
- 既有專案只補缺口，不覆蓋 `AGENTS.md`、`README.md`、`.gitignore`、Git 歷史、Firebase 設定。
- 路徑避免空格；空格路徑曾造成補丁工具與側邊欄刷新困擾。
- Finder 或 Google Drive 改名後，Codex 側邊欄不一定立即刷新。
- 專案改名時，要同步檢查 Google Drive 資料夾、`AGENTS.md`、Obsidian 專案庫、駕駛艙、`.codex/config.toml` 信任路徑與 Firebase MCP project directory。
- Firebase Project ID 不會因本地資料夾改名而自動改變。
- 不把 `.env`、API key、token、密碼、Admin 憑證、個資或敏感資料寫入 repo 或筆記。
- Obsidian 的日記同步是既有每日筆記系統的一部分，不等於 Arry 助手要建立新的 daily log。

## 新專案初始化時怎麼用

當使用者說「開始新專案」、「新專案初始化」、「初始化專案工作模式」或「建立專案」時：

1. 先執行 `07-專案初始化工作模式`。
2. 若專案要套用 Arry 助手，依本文件補上四盒結構。
3. 專案本地只建 `100_Todo/` 與 `200_Reference/`。
4. 只有使用者明確需要專案專屬助手記憶時，才加 `000_Agent/project-memory/`。
5. 專案 `AGENTS.md` 與 Obsidian 駕駛艙都要寫明：全域核心層在 `codex_installation/000_Agent`，本地資料層只放該專案自己的工作與素材。
