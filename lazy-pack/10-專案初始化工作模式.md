# 10-專案初始化工作模式

## 目標

把「開始新專案」變成固定工作模式，不只套用在單一專案。

## 前置條件

- 已完成 `README.md` 的設定表。
- 已決定 `{{WORK_ROOT}}`。
- 若使用 Obsidian，已完成 `04-建立第二大腦-Obsidian`。
- 若使用 GitHub，已完成 `03-連接-GitHub`。
- 若使用個人助手資料層，已完成 `09-個人助手設定`。

## 全域規則位置

Codex 全域規則：

```text
{{CODEX_HOME}}/AGENTS.md
```

建議加入：

```markdown
## 新專案固定工作模式

當我說「開始新專案」、「新專案初始化」、「初始化專案工作模式」、「建立專案」時：

- 先檢查環境與現有資料夾，不一開始就改檔。
- 先問清楚專案名稱、用途、工作資料夾、GitHub repo、公開或私有、是否啟用 GitHub Pages、Obsidian 駕駛艙位置、是否使用 Firebase、部署目標。
- 工作資料夾預設放在 `{{WORK_ROOT}}`。
- Obsidian 只放專案駕駛艙：`專案庫/<專案名稱>/專案工作流程.md`。
- 專案根目錄要有 `AGENTS.md`、`README.md`、`.gitignore`、Git repo。
- 若使用個人助手資料層，專案需要本地 assistant skill 或本地記憶時，建立 `000_Agent/skills/` 與 `000_Agent/memory/`；全域 Codex skills 才放 `{{CODEX_HOME}}/skills`。
- 既有專案只補缺口，不覆蓋既有設定或 Git 歷史。
- 不把 secrets 寫進 repo。
```

## 建議全域 Skills

放在：

```text
{{CODEX_HOME}}/skills
```

建議建立：

- `project-init-sync`
- `startup-sync`
- `shutdown-sync`
- `{{ASSISTANT_NAME}}-assistant` 或自己的個人助手全域入口 skill

可用 `11-Codex-Skill-Creator-工作流.md` 建立。

本懶人包已附前三個專案生命週期 skills，可直接安裝：

```bash
mkdir -p "{{CODEX_HOME}}/skills"

for skill in project-init-sync startup-sync shutdown-sync; do
  mkdir -p "{{CODEX_HOME}}/skills/$skill"
  rsync -a --delete "{{SETUP_REPO}}/lazy-pack/skills/$skill/" "{{CODEX_HOME}}/skills/$skill/"
  test -f "{{CODEX_HOME}}/skills/$skill/SKILL.md" && echo "$skill installed"
done
```

## 新專案標準流程

1. 先檢查 `{{WORK_ROOT}}` 與是否已有同名資料夾。
2. 問清楚：
   - 專案名稱
   - 用途
   - 工作資料夾
   - GitHub repo 是否需要
   - public / private
   - GitHub Pages 是否需要
   - Obsidian 駕駛艙路徑
   - Firebase 是否需要
   - 部署目標
3. 建立工作資料夾：

```text
{{WORK_ROOT}}/<project-name>
```

4. 建立或補齊：
   - `AGENTS.md`
   - `README.md`
   - `.gitignore`
   - Git repo
5. 建立 Obsidian 駕駛艙：

```text
{{OBSIDIAN_PROJECTS}}/<project-name>/專案工作流程.md
```

6. 若使用個人助手資料層，建立專案本地：

```text
100_Todo/
  drafts/
  projects/
  archive/
200_Reference/
  writing-samples/
  templates/
  past-work/
```

只有需要專案本地 assistant skill 或專案獨立記憶時，才建立：

```text
000_Agent/skills/
000_Agent/memory/
```

`000_Agent/skills` 只放該專案自己的 assistant skill、流程草稿或局部工具說明；不要 symlink 到 `{{CODEX_HOME}}/skills`。需要全域觸發的 Codex skill 才安裝到 `{{CODEX_HOME}}/skills`，並依 `15-Codex-全域-Skills-跨裝置同步` 指向 `{{SYNC_ROOT}}/skills`。
`000_Agent/memory` 只在專案真的需要獨立助手記憶時才建立。

7. 若需 GitHub，再建立 repo 與 remote。
8. 若需 Firebase，再建立或連接 project。

## 專案 AGENTS.md 範本

```markdown
# <project-name> — AGENTS.md

## 專案入口

- 專案名稱：<project-name>
- 專案用途：<purpose>
- 工作目錄：`{{WORK_ROOT}}/<project-name>`
- GitHub repo：`{{GITHUB_USER}}/<repo>` 或 `未建立`
- Obsidian vault：`{{OBSIDIAN_VAULT}}`
- 專案駕駛艙：`專案庫/<project-name>/專案工作流程.md`
- Firebase project：`{{FIREBASE_PROJECT_ID}}` 或 `未使用`

## 開工

- 讀本檔。
- 讀 Obsidian 駕駛艙。
- 檢查 Git 狀態。
- 不自動 pull、commit、push。

## 收工

- 更新 Obsidian 駕駛艙。
- 回報 Git 狀態。
- 只有使用者要求時才 commit / push。

## 安全

- 不提交 `.env`、API key、token、密碼、Admin 憑證、個資或敏感資料。
```

## 既有專案規則

若專案已存在：

- 先盤點已存在 / 缺少項目。
- 只補缺口。
- 不覆蓋既有 `AGENTS.md`、`README.md`、`.gitignore`、Firebase 設定或 Git 歷史。
- 只補上個人助手整合段落與缺少的本地 `100_Todo/`、`200_Reference/`；若該專案有本地 assistant skill 或記憶需求，再補 `000_Agent/skills/`、`000_Agent/memory/`。

## 命名規則

新專案資料夾建議使用：

- 小寫英文
- 數字
- 底線 `_`
- 連字號 `-`

避免空格。

## 驗證

1. 專案資料夾存在。
2. `AGENTS.md`、`README.md`、`.gitignore` 存在。
3. `git status --short --branch` 可執行。
4. Obsidian 駕駛艙存在。
5. 若有 GitHub，`git remote -v` 正確。
6. 若有 Firebase，`.firebaserc` 指向正確 project。

## 設定範例

本機曾使用專案名稱 `codex_installation`。這只是範例；下載者請使用自己的專案名稱。

## 踩坑修正

- 新增全域 skills 後，要重開 Codex 對話或重啟 App。
- 來源文件若提到 來源工具規則檔 或 來源工具的 skills 路徑，要先判斷層級：全域觸發才轉成 `{{CODEX_HOME}}/skills`；專案/助手本地 skill 轉成 `000_Agent/skills`。
- Codex 側邊欄不會即時跟著 Finder / 雲端硬碟改名刷新。
- 專案改名時要同步改：
  - 工作資料夾
  - `{{CODEX_CONFIG}}` 信任路徑或 MCP 路徑
  - 專案 `AGENTS.md`
  - Obsidian 專案庫資料夾
  - Obsidian 駕駛艙內文
  - Firebase MCP project directory
- GitHub 若沒有 remote，就不需要改 repo。
- Firebase Project ID 不會因本地專案改名而改變。
